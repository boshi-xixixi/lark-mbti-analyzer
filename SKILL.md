# 飞书用户 MBTI 分析助手

> **前置条件：** 先阅读 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md)。

## 功能概述

本 Skill 通过分析飞书用户的聊天记录，基于 MBTI（迈尔斯 - 布里格斯类型指标）理论，从四个维度分析用户性格：
- **E/I (外向/内向)**: 通过消息频率、互动积极性、社交倾向判断
- **S/N (感觉/直觉)**: 通过关注细节还是概念、具体还是抽象判断
- **T/F (思考/情感)**: 通过决策方式、表达风格、价值取向判断
- **J/P (判断/知觉)**: 通过计划性、灵活性、生活方式判断

## 核心命令

### 快捷命令（推荐）

```bash
# 分析指定用户的 MBTI 性格
lark-cli im +analyze-mbti --user "用户名" [--days 30] [--limit 100]

# 通过用户 ID 分析
lark-cli im +analyze-mbti --user-id "ou_xxx" [--days 30] [--limit 100]

# 分析群聊中所有活跃成员的 MBTI
lark-cli im +analyze-group-mbti --chat "群聊名称" [--days 30] [--limit 200]

# 通过群聊 ID 分析
lark-cli im +analyze-group-mbti --chat-id "oc_xxx" [--days 30] [--limit 200]
```

### 分步操作

```bash
# Step 1: 搜索用户（如果只知道姓名）
lark-cli contact +search --name "张三"

# Step 2: 获取聊天记录
# 私聊记录
lark-cli im +get-messages --user-id "ou_xxx" --days 30 --limit 100

# 群聊记录

# Step 3: 分析 MBTI（AI 会自动处理消息数据并生成报告）
# AI 会读取消息内容，应用 MBTI 分析算法，生成详细报告
```

## MBTI 分析算法

### 维度 1: E/I (外向 Extraversion / 内向 Introversion)

**E 外向特征：**
- 消息频率高，主动发起对话
- 使用积极语气词（哈哈、太棒了、赞、鼓掌等）
- 喜欢群体活动（大家一起、咱们、组织、聚会等）
- 快速回复，即时互动
- 表达热情（使用表情符号、感叹号等）

**I 内向特征：**
- 消息较少但内容有深度
- 偏好独立思考（我认为、个人觉得、我的想法是等）
- 需要思考时间（稍等、想想、考虑一下等）
- 偏好一对一沟通
- 表达谨慎（可能、也许、或许等）

### 维度 2: S/N (感觉 Sensing / 直觉 Intuition)

**S 感觉特征：**
- 关注具体事实和数据（数字、百分比、具体日期等）
- 重视实际经验（案例、例子、操作步骤等）
- 描述详细细节（具体、详细、流程等）
- 关注当下和已完成（现在、目前、已经、完成等）
- 实际落地导向（执行、实施、落地等）

**N 直觉特征：**
- 关注未来和可能性（未来、规划、愿景、潜力等）
- 讨论抽象概念（概念、理论、战略、宏观等）
- 创新思维（创意、创新、想法、想象等）
- 假设性思考（如果、假如、会不会等）
- 趋势和方向（趋势、方向、愿景、梦想等）

### 维度 3: T/F (思考 Thinking / 情感 Feeling)

**T 思考特征：**
- 逻辑分析（因为、所以、因此、由于等）
- 客观评价（应该、必须、需要、正确、错误等）
- 任务和问题导向
- 直接表达观点
- 重视效率和结果

**F 情感特征：**
- 情感表达（开心、难过、感谢、抱歉等）
- 关系维护（团队、和谐、合作、支持等）
- 关心他人感受（理解、照顾、尊重、关心等）
- 委婉表达
- 重视氛围和人情

### 维度 4: J/P (判断 Judging / 知觉 Perceiving)

**J 判断特征：**
- 计划和安排（计划、安排、时间表、截止日期等）
- 目标导向（目标、完成、结论、总结等）
- 结构化表达（第一第二第三、首先然后最后等）
- 确定性语言（确定、决定、一定、必须等）
- 时间观念强

**P 知觉特征：**
- 灵活开放（灵活、随机、开放、选择等）
- 不确定性语言（看情况、随便、都可以、不一定等）
- 过程导向（尝试、探索、体验、过程等）
- 保持选择权（或许、大概、到时候再说等）
- 享受当下

## 分析流程

### 单用户分析流程

1. **获取用户 ID**
   - 如果提供姓名：`lark-cli contact +search --name "张三"`
   - 如果提供 ID：直接使用

2. **获取聊天记录**
   ```bash
   lark-cli im +get-messages --user-id "ou_xxx" --days 30 --limit 100
   ```

3. **分析消息内容**
   - 统计各维度关键词出现频率
   - 识别表达模式和语气特征
   - 计算每个维度的得分（-1 到 +1）
   - 确定性格类型（每个维度取绝对值较大的倾向）

4. **生成分析报告**
   - MBTI 类型（如 ENFP、INTJ 等）
   - 各维度得分和置信度
   - 性格总结
   - 沟通风格
   - 优势和潜在改进点
   - 典型消息示例

### 群聊分析流程

1. **获取群聊 ID**
   - 如果提供群名：`lark-cli im +search-chat --name "项目群"`
   - 如果提供 ID：直接使用

2. **获取群聊消息**
   ```bash
   lark-cli im +get-chat-messages --chat-id "oc_xxx" --days 30 --limit 200
   ```

3. **识别活跃成员**
   - 统计每个成员的发言数量
   - 筛选发言数>5 的成员

4. **为每个活跃成员生成 MBTI 报告**
   - 提取该成员的所有消息
   - 应用单用户分析流程
   - 生成个人报告

5. **生成团队汇总报告**
   - 团队 MBTI 分布图
   - 团队性格特点总结
   - 协作建议

## 置信度说明

- **高置信度 (>70%)**: 消息数量充足（>20 条），各维度特征明显
- **中等置信度 (50-70%)**: 消息数量适中（10-20 条），特征较为明显
- **低置信度 (<50%)**: 消息较少（<10 条）或特征不明显

**建议**: 置信度低于 50% 时，建议增加分析的消息数量或时间范围。

## 注意事项

1. **数据量要求**: 至少需要 10 条有效消息才能进行准确分析
2. **时间范围**: 建议分析最近 30-90 天的消息
3. **消息质量**: 消息内容越丰富、越多样，分析越准确
4. **隐私保护**: 分析前需获得用户同意，报告应妥善保管
5. **准确性**: MBTI 分析仅供参考，不能替代专业心理测评
6. **文化差异**: 考虑中文表达习惯和文化背景
7. **语境影响**: 工作场景和私人场景的表达可能不同

## 失败处理

### 消息不足
**错误**: 未获取到足够的消息数据
**解决**: 
- 扩大时间范围（--days 60 或--days 90）
- 增加消息数量限制（--limit 200）
- 说明数据不足，提供定性分析

### 权限错误
**错误**: Permission denied, missing scope: im:message
**解决**:
- 重新登录：`lark-cli auth login --scope im:message,contact:contact`
- 检查应用权限配置

### 用户不存在
**错误**: 未找到指定用户
**解决**:
- 检查用户姓名或 ID 是否正确
- 使用 `lark-cli contact +search` 搜索确认

## 算法实现细节

### 评分计算

1. **关键词计数**:
   ```
   维度得分 = (正向关键词数 - 反向关键词数) / 总关键词数
   ```

2. **模式匹配加分**:
   - 每个匹配的模式加 0.5 分
   - 同一模式不重复计分

3. **归一化**:
   ```
   归一化得分 = (维度 2 得分 - 维度 1 得分) / (总得分 + 1)
   范围：-1 到 +1
   ```

4. **类型判断**:
   - 得分 > 0: 倾向第二个字母（E、N、F、P）
   - 得分 < 0: 倾向第一个字母（I、S、T、J）
   - |得分| > 0.5: 明显倾向
   - |得分| < 0.3: 相对平衡

### 置信度计算

```
置信度 = (
  消息数量因子 * 0.3 +
  维度差异因子 * 0.4 +
  证据丰富度 * 0.3
)
```
