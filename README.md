## 📖 项目介绍

**lark-mbti-analyzer** 是一款基于飞书 CLI 的智能 Skill，通过分析用户在飞书上的聊天记录，运用 MBTI（迈尔斯 - 布里格斯类型指标）理论，自动识别和生成详细的性格分析报告。

### 核心功能

- 🔍 **单用户分析**：分析指定用户的 MBTI 性格类型
- 👥 **群聊分析**：一次性分析群聊中所有活跃成员的性格特点
- 📊 **团队报告**：生成团队 MBTI 分布图和协作建议
- 💡 **沟通建议**：提供个性化的沟通风格分析和改进建议
- 🎯 **置信度评估**：智能评估分析结果的可信度

## ✨ 特性列表

- **基于真实聊天数据**：分析用户在飞书上的真实沟通记录，结果更贴近实际
- **四维度分析**：从 E/I、S/N、T/F、J/P 四个维度全面评估性格特征
- **智能关键词识别**：内置丰富的中文表达习惯词库，准确捕捉性格特征
- **可视化报告**：生成美观易读的详细分析报告，包含图表和典型消息示例
- **团队分析**：支持群聊分析，帮助团队了解成员性格分布
- **隐私保护**：仅在本地分析，数据不会上传到外部服务器
- **灵活配置**：支持自定义分析时间范围和消息数量

## 🚀 安装和使用方法

### 前置要求

1. **安装飞书 CLI**
   ```bash
   # 使用 npm 安装
   npm install -g lark-cli
   
   # 或使用 yarn
   yarn global add lark-cli
   ```

2. **配置飞书 CLI**
   ```bash
   # 初始化配置
   lark-cli config init
   
   # 登录授权
   lark-cli auth login --domain mbti-analyzer
   ```

3. **安装本 Skill**
   ```bash
   # 克隆仓库
   git clone https://github.com/boshi-xixixi/lark-mbti-analyzer.git
   
   # 将 SKILL.md 复制到你的 Trae skills 目录
   cp lark-mbti-analyzer/SKILL.md ~/.trae/skills/
   ```

### 使用方式

#### 方式一：快捷命令（推荐）

```bash
# 分析指定用户的 MBTI 性格
lark-cli im +analyze-mbti --user "用户名" [--days 30] [--limit 100]

# 通过用户 ID 分析
lark-cli im +analyze-mbti --user-id "ou_xxx" [--days 30] [--limit 100]

# 分析群聊中所有活跃成员的 MBTI
lark-cli im +analyze-group-mbti --chat "群聊名称" [--days 30] [--limit 200]

# 通过群聊 ID 分析
lark-cli im +analyze-group-mbti --chat-id "oc_xxx" [--days 30] [--limit 200]
```

#### 方式二：分步操作

```bash
# Step 1: 搜索用户（如果只知道姓名）
lark-cli contact +search --name "张三"

# Step 2: 获取聊天记录
# 私聊记录
lark-cli im +get-messages --user-id "ou_xxx" --days 30 --limit 100

# 群聊记录
lark-cli im +get-chat-messages --chat-id "oc_xxx" --days 30 --limit 200

# Step 3: AI 会自动分析消息数据并生成报告
```

## 🧩 MBTI 四个维度说明

### 1️⃣ E/I (外向 Extraversion / 内向 Introversion)

**能量来源维度**：你从哪里获得能量？

| 外向 (E) | 内向 (I) |
|---------|---------|
| 从与人交往中获得能量 | 从独处思考中获得能量 |
| 喜欢群体活动 | 偏好一对一交流 |
| 行动导向 | 思考导向 |
| 表达热情直接 | 表达含蓄内敛 |
| 快速反应 | 深思熟虑 |

**典型特征词**：
- E 型：哈哈、太棒了、赞、大家一起、咱们、组织
- I 型：我认为、个人觉得、稍等、想想、可能、也许

### 2️⃣ S/N (感觉 Sensing / 直觉 Intuition)

**信息收集维度**：你如何收集信息？

| 感觉 (S) | 直觉 (N) |
|---------|---------|
| 关注具体事实和数据 | 关注未来和可能性 |
| 重视实际经验 | 讨论抽象概念 |
| 描述详细细节 | 创新思维 |
| 关注当下 | 假设性思考 |
| 实际落地 | 趋势方向 |

**典型特征词**：
- S 型：数字、百分比、案例、例子、具体、详细、执行、落地
- N 型：未来、规划、愿景、概念、理论、战略、创意、创新

### 3️⃣ T/F (思考 Thinking / 情感 Feeling)

**决策方式维度**：你如何做决策？

| 思考 (T) | 情感 (F) |
|---------|---------|
| 逻辑分析 | 情感表达 |
| 客观评价 | 关系维护 |
| 任务导向 | 关心他人感受 |
| 直接表达 | 委婉表达 |
| 重视效率 | 重视氛围 |

**典型特征词**：
- T 型：因为、所以、因此、应该、必须、正确、错误
- F 型：开心、难过、感谢、团队、和谐、合作、理解

### 4️⃣ J/P (判断 Judging / 知觉 Perceiving)

**生活方式维度**：你如何组织生活？

| 判断 (J) | 知觉 (P) |
|---------|---------|
| 计划和安排 | 灵活开放 |
| 目标导向 | 不确定性语言 |
| 结构化表达 | 过程导向 |
| 确定性语言 | 保持选择权 |
| 时间观念强 | 享受当下 |

**典型特征词**：
- J 型：计划、安排、目标、完成、第一第二第三、确定、决定
- P 型：灵活、随机、开放、看情况、随便、都可以、尝试、探索

## 📊 输出示例

### 完整报告结构

一份完整的 MBTI 分析报告包含以下部分：

1. **基础信息**
   - 用户姓名和 ID
   - 分析时间
   - 消息数量和时间范围

2. **MBTI 类型**
   - 四字母类型（如 ENFP、INTJ）
   - 置信度百分比
   - 一句话性格描述

3. **四维度详细分析**
   - 每个维度的得分和置信度
   - 主要证据列表
   - 典型消息示例

4. **性格总结**
   - 综合性格描述
   - 沟通风格特点

5. **优势和改进点**
   - 2-4 个主要优势
   - 2-3 个潜在改进建议

6. **工作建议**
   - 适合的工作类型
   - 理想的工作环境
   - 协作建议

### 团队报告特色

团队报告会额外包含：

- **成员 MBTI 分布列表**：每个成员的性格类型
- **团队维度分布图**：可视化展示各维度比例
- **团队性格特点**：整体特征总结
- **协作建议**：针对性的团队合作建议

## ⚠️ 注意事项

### 数据要求

1. **最少消息数**：至少需要 10 条有效消息才能进行准确分析
2. **建议时间范围**：分析最近 30-90 天的消息效果最佳
3. **消息质量**：消息内容越丰富、越多样，分析结果越准确

### 隐私保护

- ⚠️ **分析前需获得用户同意**：尊重他人隐私权
- 📁 **报告妥善保管**：避免泄露他人性格信息
- 🔒 **仅限个人使用**：不要用于招聘、评估等敏感场景

### 准确性说明

- 📊 **仅供参考**：MBTI 分析不能替代专业心理测评
- 🌏 **文化差异**：已考虑中文表达习惯，但仍可能存在偏差
- 💬 **语境影响**：工作场景和私人场景的表达可能不同
- 📈 **置信度评估**：请关注报告中的置信度指标

### 置信度说明

| 置信度范围 | 说明 | 建议 |
|----------|------|------|
| >70% | 高置信度 | 消息充足，特征明显，结果可靠 |
| 50-70% | 中等置信度 | 消息适中，特征较明显，结果可参考 |
| <50% | 低置信度 | 消息较少或特征不明显，建议增加数据 |

## 🛠 常见问题

### 消息不足

**错误**: 未获取到足够的消息数据

**解决方案**：
- 扩大时间范围：`--days 60` 或 `--days 90`
- 增加消息数量限制：`--limit 200`
- 说明数据不足，AI 会提供定性分析

### 权限错误

**错误**: Permission denied, missing scope: im:message

**解决方案**：
```bash
# 重新登录并授权
lark-cli auth login --scope im:message,contact:contact,im:chat
```

### 用户不存在

**错误**: 未找到指定用户

**解决方案**：
- 检查用户姓名或 ID 是否正确
- 使用 `lark-cli contact +search` 搜索确认

### 分析结果不准确

**可能原因**：
- 消息数量不足
- 用户表达风格特殊
- 语境单一（如仅工作场景）

**解决方案**：
- 增加分析的消息数量和时间范围
- 结合多个场景的消息
- 将结果作为参考，而非绝对判断
